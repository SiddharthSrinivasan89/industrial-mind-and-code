The data is synthetic but calibrated to reflect realistic Indian
automotive OEM dispatch patterns: FY-end push, monsoon slowdown,
and festive season surge. Each vehicle dispatched requires a set of
lighting assemblies (headlamps, tail lamps, DRLs), serving as the
production-driven demand signal for the component supply chain.


2. SUPPLY CHAIN STRUCTURE
-------------------------
The simulation models a 3-tier auto ancillary component supply chain:

  Tier 1 — OEM Assembly Plant (role key: "oem")
    Tatva Motors vehicle assembly. Sees monthly dispatch targets
    (production demand). Orders lighting assemblies from Tier 1 supplier.

  Tier 2 — Lighting Manufacturer (role key: "ancillary")
    Auto ancillary lighting assembler.
    Sees OEM's lighting orders. Orders LED modules and optical
    components from Tier 2 supplier.

  Tier 3 — LED/Component Supplier (role key: "ancillary_supplier")
    LED module and lighting component manufacturer.
    Sees lighting manufacturer's component orders.
    Decides production volume.

Information flow: Only the OEM tier sees the actual dispatch demand.
Each upstream tier sees only orders from its immediate downstream
partner, creating the classic information asymmetry that drives the
bullwhip effect through the component supply chain.

Why lighting? Automotive lighting assemblies are:
  - Common to ALL vehicles (ICE, EV, hybrid) — no powertrain bias
  - Always outsourced to auto ancillaries (never made in-house by OEM)
  - Large, complex assemblies with their own sub-component supply chain
  - A well-established Indian auto ancillary sector


3. EXPERIMENT PARAMETERS
------------------------
Parameter               Value       Justification
-----------------------------------------------------------------------
ordering_periods        13          Matches 13 months of data
lead_time_periods       1           Typical monthly ordering cycle
                                    (order this month, receive next month)
initial_inventory       180,000     ~4 months of average production
                                    (~46K/month), buffer against seasonal
                                    spikes without being excessive
time_unit               month       All Indian auto industry ordering
                                    operates on monthly cycles
runs_per_config         10          Statistical significance for OVAR
                                    calculation across stochastic LLM runs
random_seed             42          Reproducibility of demand sequence
forecast_horizon        3           OEM sees 3 months ahead; upstream
                                    tiers get no direct demand visibility
max_order_multiplier    5.0         Ceiling clamp: orders capped at 5x
                                    current demand to prevent runaway
order_floor             0.2x        Floor: minimum order = 20% of demand


4. LLM AGENT CONFIGURATIONS
----------------------------
Two model tiers:

  Lightweight (gpt-4.1-mini)
    Temperature:        0.2
    Max tokens:         600
    Inter-call delay:   1.0 seconds
    Use case:           Fast iteration, baseline experiments

  Reasoning (o1)
    Temperature:        1.0 (ignored by o1, kept for schema)
    Max tokens:         16,000
    Inter-call delay:   5.0 seconds
    Use case:           Deep reasoning about production patterns

Two agent categories:

  Blind Agent
    Sees: inventory, backlog, in-transit orders, demand number, history
    Does NOT see: dates, geography, product category, market context
    Role framing: Generic OEM/ancillary/supplier (no India context)
    Expected behavior: Reactive ordering, higher bullwhip amplification

  Context Agent
    Sees: Everything the blind agent sees, PLUS:
      - Month name and year
      - Product (lighting for Tatva Volta, Prism, Horizon, Savana, Arc)
      - Market context (India, automotive component supply chain)
      - Role framing (Tatva Motors OEM, Tier 1 ancillary)
    Does NOT see: Explicit pattern labels (never told "Navratri" or "monsoon")
    Expected behavior: Infers seasonal production patterns from world
      knowledge, potentially dampening bullwhip through anticipatory ordering


5. KEY METRICS
--------------
OVAR (Order Variance Amplification Ratio)
  Formula:  Var(orders placed) / Var(demand received)
  Per tier, per run. OVAR > 1 indicates bullwhip amplification.

Stockout periods:  Count of months with zero available inventory
Inventory levels:  End-of-period inventory at each tier
Order patterns:    Time series of orders placed by each tier


6. SEASONAL PATTERNS IN THE DATA
---------------------------------
The synthetic data embeds realistic Indian automotive patterns that
propagate through the component supply chain:

  FY-end push (March):     49,287 — pre-registration rush drives
                           increased production and component orders
  Summer slowdown (May-Jun): 39,841 / 36,478 — reduced production
                           as dispatches decline seasonally
  Monsoon dip (Jul):       38,193 — further production slowdown
  Festive surge (Sep-Nov): 56,892 / 59,608 / 55,347 — Navratri,
                           Dussehra, Diwali drive peak production;
                           component suppliers face surge orders
  Year-end (Dec):          48,491 — post-festive normalization

The context agent is never explicitly told about these patterns. The
experiment tests whether LLM world knowledge about Indian automotive
seasonality helps component suppliers anticipate production shifts and
reduce order variance amplification through the supply chain.


7. LIMITATIONS
--------------
- Small sample: 13 data points limits statistical power
- Synthetic data: Based on realistic patterns but not actual OEM figures
- Proxy demand: Vehicle dispatches used as proxy for component orders;
  real OEM-to-ancillary ordering may batch or smooth differently
- 1:1 ratio assumed: Each vehicle = 1 lighting assembly set; real
  ratios may vary by model (different headlamp/tail lamp specs)
- Simplified supply chain: Real ancillary supply chains have multiple
  OEM customers, shared production lines, and model-specific tooling
- LLM stochasticity: Even with low temperature, LLM responses vary
  across runs, requiring multiple runs per configuration
- No capacity constraints: The simulation allows unlimited production;
  real ancillary factories have capacity ceilings and ramp-up times


8. EXPERIMENT MATRIX
--------------------

  ---------------------------------------------------------------
